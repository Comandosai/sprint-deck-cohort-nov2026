1 Отдельная ботовая Gmail (3 мин)

Почему отдельная. Боту даём доступ к почте и календарю. Если использовать личную — бот через ошибку может удалить важное письмо или отправить от твоего имени. Отдельный аккаунт = чистая граница.

  1. Открой инкогнито-окно в Chrome/Safari (важно: не мешать своему Google-аккаунту)
  2. accounts.google.com → «Create account» → «For my personal use»
  3. Имя любое (например Openclaw Bot)
  4. Username: <твоё-имя>-openclaw@gmail.com
  5. Пароль → 2FA по желанию
  6. Сохрани логин/пароль в менеджер паролей — без этого через месяц забудешь и потеряешь доступ к боту

2 Подписка ChatGPT Plus $20/мес (2 мин)

Зачем. В3 переходим на GPT-5 как primary модель бота. Доступ через Codex CLI OAuth — официальный способ OpenAI. Подписка даёт безлимит на GPT-5 в боте.

  1. chat.openai.com → залогинься
  2. Профиль слева снизу → «Upgrade plan» → Plus → оплатить

Если уже есть Plus — пропусти.

3 Brave Search API ключ (3 мин)

Зачем. Бесплатные 2000 поисковых запросов/мес. Это web_search для бота — когда нужно «загуглить».

  1. brave.com/search/api/ → «Get Started»
  2. Зарегистрируйся (email)
  3. Subscribe → «Free plan» 2000 queries/month → без карты
  4. Dashboard → API Keys → скопируй ключ (формат BSA...)
  5. Сохрани как BRAVE_API_KEY=BSA...

4 Tavily API ключ (3 мин)

Зачем. Второй поисковик для глубокого ресёрча. Возвращает структурированные данные специально для AI-агентов (а не HTML как Brave). Free tier 1000 запросов/мес.

  1. tavily.com → Sign up
  2. Dashboard → копируй API key (tvly-...)
  3. Сохрани как TAVILY_API_KEY=tvly-...

5 Google Cloud Console — самое сложное (12 мин)

Зачем. Чтобы бот мог читать Gmail и Calendar — нужен OAuth. Настраивается в Google Cloud Console. На выходе получишь файл client_secret.json.

КРИТИЧНО: весь этот шаг делается под ботовой Gmail из шага 1, не под личным аккаунтом!

Часть 5.1 — Создать проект
  1. Инкогнито-окно → console.cloud.google.com
  2. Залогинься ботовой почтой
  3. Accept Terms (если первый раз)
  4. Сверху dropdown «Select a project» → NEW PROJECT
  5. Name: openclaw-bot, Location: «No organization»
  6. CREATE → ждёшь 30 сек → Switch to project
Часть 5.2 — Включить 2 API
  1. Левое меню ☰ → APIs & Services → Library
  2. Поиск «Gmail API» → ENABLE
  3. Назад в Library → «Google Calendar API» → ENABLE
⚠️
Интерфейс Google Cloud Console обновился (2025-2026). В новых проектах раздел «OAuth consent screen» часто называется «Google Auth Platform», а внутри него отдельные пункты: Branding / Audience / Clients / Data Access. Шаги ниже учитывают и старый и новый интерфейс.
Часть 5.3 — OAuth Consent Screen / Branding
  1. ☰ → APIs & Services → OAuth consent screen (или Google Auth Platform → Branding)
  2. Если первый раз — Get Started
  3. App name: openclaw-bot
  4. User support email + Developer contact: твоя ботовая почта
  5. User Type: External
  6. Save and Continue
Часть 5.4 — Data Access (бывшие Scopes!)

Это бывшие Scopes — название изменилось на «Data Access».

  1. В разделе OAuth consent screen / Google Auth Platform слева ищи пункт «Data Access»
  2. Add or Remove Scopes
  3. В фильтре «gmail» → отметь чекбоксами:
    • .../auth/gmail.readonly
    • .../auth/gmail.send
  4. В фильтре «calendar» → отметь:
    • .../auth/calendar.events.readonly
    • .../auth/calendar.readonly
  5. Update → Save
Часть 5.5 — Audience (Test Users внутри!)

Test Users больше не отдельная страница — они внутри пункта «Audience». Без них OAuth выдаст «access blocked».

  1. В разделе OAuth consent screen / Google Auth Platform слева → «Audience»
  2. Прокрути вниз до секции «Test users»
  3. + Add users → введи свою ботовую почту
  4. ADD → Save
Часть 5.6 — Credentials (ОТДЕЛЬНЫЙ пункт меню!)

Credentials — это отдельный пункт меню, НЕ внутри OAuth consent screen. Соседний.

  1. ☰ → APIs & Services → Credentials (рядом с OAuth consent screen — не внутри!)
  2. Сверху + Create Credentials → OAuth client ID
  3. Application type: Desktop app
  4. Name: openclaw-bot-desktop → CREATE
  5. В модалке нажми Download JSON → файл client_secret_XXXX.json в Downloads
  6. Открой файл в TextEdit и сохрани в заметках:
    • GOOGLE_OAUTH_CLIENT_ID= (значение client_id)
    • GOOGLE_OAUTH_CLIENT_SECRET= (значение client_secret)
  7. Сам файл client_secret_XXXX.json НЕ удаляй — он понадобится на В3

6 💡 DataImpulse прокси — ОПЦ (5 мин)

Когда нужно: если работаешь с сайтами которые геоблокируют (booking.com) или агрессивно банят датацентровые IP (Cloudflare-защищённые).

Когда НЕ нужно: простой поиск, чтение статей, бронирование — Patchright + Xvfb справятся без residential proxy.

  1. dataimpulse.com → Sign Up
  2. Top Up минимум $1 (1 GB трафика — на месяцы для бота)
  3. Dashboard → Residential proxy → копируй Username и Password
  4. Сохрани как PROXY_USER=... и PROXY_PASS=...
Готов? Открывай Установку

Шаг A — SSH на VPS (флаг -t ОБЯЗАТЕЛЕН для TTY):

Mac Terminal · в папке проекта
set -a && source .env && set +a && ssh -t -i ~/.ssh/clawd_ed25519 clawd@$VPS_IP

Если ключ называется иначе — посмотри ls ~/.ssh/ и подставь имя. Если в .env нет VPS_IP — добавь строку VPS_IP=твой.ip.адрес.

✅ Как понять что ты на VPS: приглашение поменяется с username@Mac % на clawd@<имя-vps>:~$. Без -t SSH войдёт и сразу выйдет без сессии.

Шаг C — запусти OpenClaw OAuth flow (одна команда!):

VPS · после ssh
