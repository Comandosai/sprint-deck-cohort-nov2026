Блок 01 · Интернет

Настраиваем 4 способа работы с интернетом — от быстрого поиска до настоящего браузера на VPS.

Как это работает

1
web_search — Поиск сниппетов
Бот ищет в Brave Search и получает заголовки + короткие выдержки. Как Google, только в чате.
Бесплатно · 2 000/мес
2
web_fetch — Скачать страницу
Бот скачивает полный текст страницы и читает его. Быстро, для обычных сайтов.
Бесплатно
3
Tavily MCP — AI-поиск
Глубокий поиск с AI-ранжированием — умнее Brave, сам отфильтровывает мусор. Подключим в Блоке 03.
API · 1 000/мес free
4
browser — Настоящий Chrome на VPS
Когда сайт на React/Vue и web_fetch вернул пустую страницу — бот запускает Chrome, кликает, читает.
Тяжело · нужен Chromium
🔑
Шаг 1 · Получить Brave Search API ключ
Регистрация на Brave
Зачем: Brave — бесплатный поисковик без трекинга. Free tier даёт 2 000 запросов в месяц — этого хватает на одного человека. Ключ нужен, чтобы OpenClaw мог делать запросы от твоего имени.
1
Открой сайт и зарегистрируйся
brave.com/search/api → «Get Started for Free» → регистрация через email. Карта не нужна.
2
Скопируй API ключ
После регистрации появится ключ — он начинается с BSA_. Скопируй его.
3
Добавь ключ в файл .env на VPS
Открой файл ~/.openclaw/.env и добавь строку: BRAVE_API_KEY=BSA_твой_ключ
⚙️
Шаг 2 · Настроить tools и browser в конфиге
Конфиг openclaw.json
Зачем: OpenClaw по умолчанию не знает, какие инструменты разрешены и какой поисковик использовать. Мы добавляем секцию tools — она говорит боту: «используй Brave для поиска, и вот как настроить браузер».
1
Попроси агента добавить секцию tools в конфиг
Агент сам откроет ~/.openclaw/openclaw.json и добавит нужный блок по промпту из гайда.
2
Проверь что браузер работает
Одна команда диагностики — openclaw browser doctor --deep — скажет есть ли Chromium на VPS.
3
Если Chromium не нашёлся — запусти Docker-контейнер
Одна команда docker run поднимает Browserless — изолированный Chrome. Потом указываешь его адрес в конфиге.
Проверка
Спроси бота: «найди 3 свежие новости про Anthropic». Если вернул реальные ссылки — работает.

Блок 02 · Проактивность

Настраиваем HEARTBEAT.md — файл с проверками, который бот читает каждый час — и 4 автоматических расписания.

Что бот будет делать сам

09:00 · каждый день
Утренний брифинг
Встречи дня, важные письма за ночь, погода, топ AI-новость, 3 главные задачи
21:00 · каждый день
Вечерний дайджест
Что закрыл за день, что перенёс на завтра, сколько потратил на API
09:00 · понедельник
Недельный обзор
Статистика задач за неделю, расходы, топ-3 достижения, что стоит прекратить делать
11:00 · Вт / Чт / Сб
Контент-напоминание
Что запланировано на сегодня в контент-плане или 2 идеи поста из артефактов недели
ℹ️ Как работает heartbeat: каждые 60 минут бот просыпается в изолированной сессии, читает HEARTBEAT.md и проверяет 7 вещей. Если ничего важного нет — молчит. Если нашёл — присылает одно сообщение с буллетами.
📄
Шаг 1 · Создать файл проверок
HEARTBEAT.md
Зачем: HEARTBEAT.md — это «инструкция для совести». Бот читает его в каждый тик и знает что проверить: дедлайны, письма, расходы API, зависшие задачи. Файл лежит в папке профиля агента.
1
Узнай где папка профиля агента
Команда: openclaw config show | jq '.agentDir' — покажет путь.
2
Попроси агента создать файл HEARTBEAT.md
Агент создаст шаблон с 7 проверками по промпту. Ты можешь потом добавить свои специфические проверки.
3
Настрой heartbeat в openclaw.json
Добавляем секцию: каждый час, изолированная сессия, дешёвая модель, активные часы 08:00–22:00 по Москве.
🗓️
Шаг 2 · Зарегистрировать 4 расписания
Cron-задачи
Зачем: Heartbeat проверяет всё каждый час, но не форматирует отчёты. Cron-задачи — это запланированные задания: в точное время бот запускает отдельную сессию, собирает нужную инфу и шлёт тебе готовое сообщение в Telegram.
1
Узнай свой Telegram chat_id и номера топиков
Напиши боту, потом открой api.telegram.org/bot<TOKEN>/getUpdates — там будет chat_id. Если используешь форум-группу — там же найдёшь topic ID.
2
Попроси агента зарегистрировать 4 cron-задачи
Агент выполнит 4 команды openclaw cron add с нужными расписаниями, моделью и chat_id.
3
Протестируй — запусти утренний брифинг прямо сейчас
Команда openclaw cron run morning-brief запустит задачу немедленно, не дожидаясь 9:00.
Проверка
В Telegram пришло сообщение с реальными данными — встречи, задачи, цифры. Не «всё ок», а конкретика.
⚠️ Важно про часовой пояс: VPS обычно в UTC. Без явного tz: "Europe/Moscow" утренний брифинг придёт в 12:00 МСК вместо 9:00. Агент прописывает это автоматически.

Блок 03 · MCP-серверы

Подключаем бота к 4 реальным сервисам. Один раз настроил — бот работает с ними прямо из чата.

Что подключаем и зачем

🔍
Tavily
AI-поиск — умнее Brave, сам анализирует источники. Нужен для глубокого исследования.
Нужен API-ключ
📝
Notion
Бот читает и создаёт страницы в твоих базах Notion. CRM, задачи, базы знаний — всё в чате.
Нужен API-ключ
🐙
GitHub
Создаёт Issue, смотрит PR, ищет по коду. Управление репозиториями из Telegram.
Нужен PAT-токен
🎭
Playwright
Браузер через MCP — навигация, клики, скриншоты. Для автоматизации сайтов.
Без ключа
📦
Шаг 1 · Установить uvx
Пакетный менеджер для Python-MCP
Зачем: Tavily — Python-пакет, для него нужен uvx — это быстрый запускатор от команды Astral. Стартует за 200мс, кеширует пакеты. Без него Tavily не запустится.
1
Установи uvx одной командой на VPS
curl -LsSf https://astral.sh/uv/install.sh | sh — потом переоткрой терминал.
2
Проверь что установилось
uvx --version должен вернуть номер версии.
🔑
Шаг 2 · Получить 3 API-ключа
Tavily + Notion + GitHub
Зачем: каждый сервис требует ключ — это как пропуск. Ключи хранятся в файле .env на VPS и никогда не попадают в конфиг открытым текстом.
1
Tavily — зарегистрируйся и возьми ключ
tavily.com → Sign Up → API Keys. Free tier: 1 000 запросов/месяц. Ключ начинается с tvly-
2
Notion — создай Internal Integration
В Notion: Settings → Connections → Develop or manage integrations → New integration. Скопируй «Internal Integration Secret» — начинается с secret_
3
GitHub — создай Personal Access Token (Classic)
GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate. Отметь scopes: repo и workflow. Ключ начинается с ghp_
4
Добавь все три ключа в ~/.openclaw/.env
Три строки: TAVILY_API_KEY=, NOTION_API_KEY=, GITHUB_PERSONAL_ACCESS_TOKEN=
🔌
Шаг 3 · Подключить MCP-серверы
openclaw mcp set × 4
Зачем: команда openclaw mcp set регистрирует сервер — OpenClaw знает как его запустить и какие инструменты он даёт боту. Потом эти инструменты появляются в системном промпте автоматически.
1
Попроси агента подключить 4 MCP-сервера
Агент выполнит 4 команды openclaw mcp set по промпту. Playwright потребует отдельно: npx playwright install chromium
2
Для Notion — открой доступ к своим базам
В Notion: зайди в каждую базу → ··· → Connections → добавь свою интеграцию. Без этого бот не видит страницы.
3
Проверь статус всех серверов
openclaw mcp list — все 4 должны быть в статусе connected
Проверка
Спроси бота: «найди через Tavily новости про MCP за май 2026». Если ответил с реальными ссылками — Tavily работает.
🚨 Типичные ошибки с пакетами:
Нет такого пакета: @microsoft/mcp-server-playwright
Устарел (deprecated): @modelcontextprotocol/server-github
Правильно: @playwright/mcp · GitHub подключаем через URL без установки

Блок 04 · Самопрограммирование

Бот умеет создавать новые навыки сам — прямо в процессе разговора с тобой. Никакого кода.

ℹ️ Что такое скилл: это YAML-файл с инструкцией — что делать при определённом триггере. Агент сам пишет этот файл, file-watcher подхватывает его без перезапуска OpenClaw. Скилл работает сразу.

Skill vs MCP — когда что

S
Скилл — свой воркфлоу, нет готовой интеграции
Живёт в ~/.openclaw/skills/. Пример: «собери стендап из git-лога и todo.md и пришли мне».
M
MCP — есть готовый сервер для внешнего сервиса
Notion, GitHub, Tavily — всё уже написано, просто подключаешь. Не нужно изобретать.
📦
Шаг 1 · Установить плагин
self-improving-agent
Зачем: этот ClawHub-плагин даёт боту умение создавать и редактировать собственные скиллы. Без него агент понимает запрос «добавь себе навык», но не умеет его оформить в файл.
1
Установи плагин одной командой
openclaw skills install self-improving-agent
2
Проверь что установился
openclaw skills list --installed — плагин должен быть в списке со статусом active
✨
Шаг 2 · Создать первый кастомный скилл
Просим бота напрограммировать себя
Зачем: это момент, когда бот становится по-настоящему твоим. Ты описываешь нужное поведение словами — бот пишет YAML-файл, сохраняет и сразу может использовать новый навык.
1
Попроси бота создать скилл «ежедневный стендап»
Опиши что должно происходить: читать git-лог за 24 часа + todo.md → сформировать стендап из 3 пунктов → прислать в Telegram.
2
Убедись что файл создан
Проверь: ls ~/.openclaw/skills/ — там должен появиться YAML-файл.
3
Запусти скилл — напиши слово-триггер
Напиши боту то слово, которое ты попросил его использовать как триггер (например «стендап»). Скилл должен запуститься.
Проверка
Файл скилла создан + бот реагирует на триггер + openclaw skills list --installed показывает новый скилл.
💡 Идеи для своих скиллов: «когда пишу "погода" — покажи прогноз на сегодня» · «когда пишу "ревью" + путь — проверь файл на очевидные ошибки» · «когда пишу "идеи" — предложи 3 темы поста из артефактов недели»
