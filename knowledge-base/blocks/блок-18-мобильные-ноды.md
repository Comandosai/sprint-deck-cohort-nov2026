# Блок 18: Мобильные ноды (камера, GPS, голос)

> **Что:** превращаем смартфон (Android/iOS) в полноценный нод OpenClaw — агент получает доступ к камере, микрофону, GPS, экрану и push-уведомлениям.
> **Зачем:** «Сфотографируй, что передо мной» / «Где я сейчас?» / «Запиши голосовую заметку и положи в Obsidian» — задачи, недоступные без сенсоров телефона.
> **Время:** 30–45 минут (Android короче, iOS длиннее из-за TestFlight + ограничений фоновой работы).

---

## 🎯 Цель блока

К концу блока у Дмитрия:

1. На минимум одном телефоне (Android **или** iOS) установлено приложение **OpenClaw Node**.
2. Телефон спарен с gateway через **mDNS/Bonjour** (LAN) **или Tailscale Serve** (cellular/удалённо).
3. Запрос на pairing подтверждён через `openclaw devices approve <requestId>`.
4. Выданы runtime-разрешения: камера, микрофон, GPS, нотификации.
5. На Android отключена оптимизация батареи и работает foreground-service.
6. Тестовый запрос «Сфотографируй, что передо мной и опиши» проходит end-to-end (фото → gateway → LLM → ответ).
7. (Опционально) iPhone Action Button или iOS Shortcut вызывает агента.

Этот блок — фундамент для **Блока 20 (голосовой интерфейс)**, потому что микрофон поднимается именно здесь.

## ⚡ Что нового в апреле 2026

По релиз-ноутам **2026.4.25 / 2026.4.26** (см. источники):

- **QR-pairing для iOS и Android** — команда `openclaw qr` рендерит QR с `bootstrapToken`, телефон сканирует и сразу подключается. Раньше нужно было руками вбивать host:port + token.
- **Talk Mode background listening (iOS)** — частичная работа в фоне через silent push + location события (всё ещё best-effort из-за iOS suspend).
- **Voice Wake trigger-based routing** — голосовая фраза («Hey Claw») может маршрутизироваться на конкретного агента/сессию.
- **Push notifications (FCM/APNs)** — gateway теперь умеет «будить» спящие ноды через push, что лечит Android Doze и iOS suspend (фича обсуждается в issue #30138; стабильна на Android, на iOS — alpha). [VERIFY: статус в проде на момент 29 апреля 2026]
- **Расширенный Android-набор команд**: добавились `contacts`, `calendar`, `health`, `motion`, `notification.actions`. До 34+ команд по разным источникам.
- **Hard-block `ws://` к Tailscale IP** (CLI) — после регрессии в v2026.2.19 теперь требуется `wss://` для tailnet-маршрутов; `ws://` оставлен только для приватного LAN.
- **Share Extension на iOS** — отправка ссылок/фото из системного share menu прямо в OpenClaw-сессию.
- **Android v2026.2.24** — официальная версия companion-app, заявленная в Play Store-источниках. iOS — пока **internal preview / TestFlight invite-only** (по docs.openclaw.ai/platforms/ios).

## 💡 Лайфхаки и про-приёмы

1. **Tailscale > mDNS для production.** mDNS работает только в одной L2-сети, ломается на enterprise WiFi с client isolation, гостевых сетях, и cellular. Tailscale Serve даёт стабильный `wss://gateway.<tail-id>.ts.net` — телефон достаёт VPS из метро. Включается за 3 минуты, бесплатен на 1 пользователя.

2. **Сначала pairing на LAN, потом миграция на Tailscale.** Парь в первый раз, когда телефон и gateway в одной WiFi (mDNS, `ws://`). После успешного approve меняешь URL в приложении на `wss://...ts.net` — credentials остаются те же, не нужно ре-аппрувить.

3. **`openclaw qr --json` вместо `openclaw qr`.** JSON-вывод даёт `setupCode` отдельно — можно скопировать в текстовое поле приложения, если камера телефона не видит QR (yellow tint на сломанном экране и т.п.).

4. **Bootstrap-токен короткоживущий** — если QR провисел >5 минут, перегенерируй (`openclaw qr` снова). Не путай setupCode с gateway-token: первый одноразовый и со scope только под pairing.

5. **Foreground-service нотификация — это не баг, это Android API contract.** Не пытайся её скрыть — Google убьёт приложение в Android 14+. Лучше сделать её информативной («OpenClaw Node connected, gateway: home.ts.net»).

6. **Battery optimization — отключай ВСЕГДА.** Settings → Apps → OpenClaw Node → Battery → Unrestricted. Doze-режим переводит сокет в standby через 30 минут idle и отрубает foreground service после 1 часа screen-off на некоторых OEM-прошивках (Xiaomi, Huawei, Samsung One UI печально знамениты).

7. **Xiaomi/MIUI lock screen = autostart.** Помимо battery optimization, на Xiaomi/Redmi нужно отдельно включить «Autostart» и «No restrictions on background activity» в Security app. Иначе OpenClaw умирает при блокировке экрана. На Samsung — добавить в «Never sleeping apps».

8. **Android 14+ FOREGROUND_SERVICE_MICROPHONE.** Если используешь Talk Mode (микрофон в фоне), приложение должно декларировать `FOREGROUND_SERVICE_MICROPHONE` в манифесте + получить permission. Иначе ОС крашит сервис при backgrounding. [VERIFY: проверь в About-секции, что декларация есть; иначе бери community-fork с фиксом]

9. **iOS Talk Mode — best-effort, не надейся как на Android.** iOS жёстко suspend'ит app через ~30 секунд, фоновое аудио разрешено только при активном `AVAudioSession` категории playAndRecord. Воркэраунд: silent push wake-up + Live Activity для visual hint. Реалистично для iOS — Voice Wake запускать только когда телефон активно используется или подключён к зарядке.

10. **Strip EXIF + GPS перед отправкой LLM.** OpenClaw `camera.snap` присылает base64 JPEG; у фото с iPhone/Android по умолчанию EXIF содержит GPS-координаты, модель устройства, дату-время. Если LLM-провайдер хранит логи — это утечка геолокации. На VPS добавь pre-processing через `exiftool -all=` на gateway-стороне, до отправки в Claude/OpenAI.

11. **Несколько устройств одного юзера.** `openclaw config set tools.exec.node "<id-or-name>"` — назначает дефолтный нод. Иначе при `system.run` без явного `--node` агент может постучаться не на тот телефон. Имена назначай человеческие: `iphone-15`, `pixel-home`, `tablet-kitchen`.

12. **iPhone Action Button → Shortcut → OpenClaw.** На iPhone 15 Pro+ забинди Action Button на shortcut «Ask OpenClaw», который через webhook (URL Action) шлёт `POST https://gateway.<tail>.ts.net/api/agent/ask` с `{"text": "Спроси у Claude X"}`. Это работает быстрее и стабильнее, чем встроенный Voice Wake (который в iOS suspend'ится).

13. **Apple Watch / iPad — не поддерживаются нативно** (на 29.04.2026, по docs.openclaw.ai/platforms/ios). Воркэраунд для Watch: WatchOS Shortcuts → trigger через iPhone Shortcut. Для iPad — устанавливаешь iPhone-app в режиме совместимости через TestFlight.

14. **Privacy Mode для GPS.** `location.get` принимает `precision: "low"` — тогда возвращается точность до квартала, не до 5 метров. Используй для агента, который не должен знать твой точный адрес («Какая у меня сейчас погода?» хватит и low-precision).

15. **`openclaw devices reject` — для безопасности.** Если на approve-листе появился неизвестный requestId (не твой телефон) — `openclaw devices reject <requestId>` блокирует попытку. Лог пары в `~/.openclaw/audit.log`. [VERIFY: точное имя файла лога]

16. **Wake word работает только на macOS/iOS.** На Android до апреля 2026 был только PTT (push-to-talk) или continuous Talk Mode без локального wake-word detection. Issue #30447 трекает «Hey Claw» для Android — статус ещё в work. Если нужно «всегда слушает» на Android — используй community-app `yuga-hashimoto/openclaw-assistant`.

17. **Push notifications для wake.** Если включён FCM/APNs (Issue #30138), gateway может «разбудить» нод сообщением «agent X needs you». Это решает проблему с Doze: telephone просыпается на push, открывает WebSocket, выполняет команду, спит дальше.

## 📋 Готовые команды и конфиги

### Шаг 1. Подготовка gateway (на VPS)

```bash
# Проверить, что gateway слушает не только loopback
openclaw config get gateway.bind
# Должно быть "lan" или "0.0.0.0:18789", иначе:
openclaw config set gateway.bind lan
openclaw restart

# Если используем Tailscale:
tailscale serve --bg --https=443 http://localhost:18789
# Проверить:
tailscale serve status
# Получить адрес для wss://:
tailscale status --json | jq '.Self.DNSName'
# → openclaw-gw.tailABCD.ts.net
```

### Шаг 2. Сгенерировать QR

```bash
# Локальный LAN (mDNS, ws://):
openclaw qr

# Через Tailscale (wss://):
openclaw qr --remote

# Только setup-code (если QR не сканируется):
openclaw qr --setup-code-only

# Полный JSON:
openclaw qr --json
```

Вывод содержит:
- `setupCode` — короткий код (типа `LCRH-9KF2-MNXA`)
- `gatewayUrl` — `wss://...ts.net` или `ws://192.168.1.x:18789`
- `bootstrapToken` — короткоживущий, scoped только под pairing
- `urlSource` — `lan`/`tailscale`/`manual`

### Шаг 3. Установка на Android (официально, через Play Store)

1. Play Store → «OpenClaw Node» → install (`com.coderred.andclaw` v2026.2.24).
2. Открыть → разрешения runtime: Camera, Microphone, Location (Always), Notifications.
3. **Battery optimization off:** Settings → Apps → OpenClaw → Battery → Unrestricted.
4. **(Xiaomi/Huawei)** Security app → Autostart → enable, Background activity → No restrictions.
5. В приложении: Settings → Pair → Scan QR (камера) или Manual entry (paste setupCode).
6. После сканирования телефон отправит pairing-request на gateway.

### Шаг 3b. Установка на iOS (TestFlight)

1. Запросить TestFlight-инвайт через GitHub issue (`#21183 / #9121`).
2. Открыть приглашение в Mail → install TestFlight → install OpenClaw.
3. Permissions: Camera, Microphone, Location (Always), Notifications, Local Network (важно для Bonjour!).
4. Settings → Background App Refresh → On (обязательно для wake-on-push).
5. В приложении: Pair → Scan QR.

### Шаг 4. Approve на gateway

```bash
# Видим список pending:
openclaw devices list

# Пример вывода:
# requestId            device              ip              status
# req_a9f3...          iphone-15-dmitriy   100.86.4.12     PENDING

# Approve конкретный:
openclaw devices approve req_a9f3...

# Или последний:
openclaw devices approve --latest

# Проверить, что нод подключился:
openclaw nodes status
# Должно быть "online" + список capabilities (camera, location, microphone, ...)
```

### Шаг 5. Конфиг разрешённых команд

```bash
# По умолчанию camera.snap, camera.clip, screen.record — заблокированы.
# Включаем явно:
openclaw config set gateway.nodes.allowCommands \
  '["camera.list","camera.snap","location.get","screen.snapshot","microphone.record"]'

# Для запрета чего-то:
openclaw config set gateway.nodes.denyCommands '["sms.send","contacts.export"]'

# denyCommands всегда побеждает allowCommands.

# Дефолтный нод для system.run (если их несколько):
openclaw config set tools.exec.node "iphone-15-dmitriy"
```

### Шаг 6. Тест из CLI

```bash
# Photo:
openclaw nodes camera snap --node iphone-15-dmitriy --facing back --quality 0.9
# → возвращает base64 + сохраняет в ~/.openclaw/captures/

# Video clip:
openclaw nodes camera clip --node iphone-15-dmitriy --facing back \
  --duration 10000 --include-audio false

# Location:
openclaw nodes location get --node iphone-15-dmitriy --precision low
# → { "lat": 55.75, "lon": 37.61, "accuracy": 800, "ts": "..." }

# Status всех нодов:
openclaw nodes status --json | jq '.[] | {name, online, caps}'
```

### Шаг 7. Тестовый запрос агенту

В Telegram (или через Control UI):

> Сфотографируй заднюю камеру моего iPhone-15 и опиши, что на фото.

Агент должен:
1. Вызвать tool `camera.snap` с `node: iphone-15`, `facing: back`.
2. Получить base64 JPEG.
3. Передать его в LLM (multimodal) с промптом «describe this image».
4. Ответить пользователю.

## ⚠️ Подводные камни

1. **mDNS не работает через Wireguard/Tailscale без `tailscale serve`.** Bonjour — это link-local multicast (224.0.0.251), не маршрутизируется через VPN-туннели. Если телефон в cellular — только `wss://...ts.net` через `tailscale serve`.

2. **`ws://` к Tailscale IP заблокирован CLI с v2026.2.19** (issue #21760). Нельзя написать `ws://100.x.y.z:18789`. Только `wss://<peer>.ts.net` через `tailscale serve --https=443`. Регрессия частично откачена, но для production используй `wss://`.

3. **Android Doze убивает foreground service на OEM-прошивках.** AOSP-Android (Pixel) держит сокет нормально. MIUI/EMUI/One UI агрессивно убивают. Решение: battery unrestricted + autostart + AccessibilityService «keep awake» как страховка, либо переход на push-wake (FCM).

4. **iOS Background Modes — белый список, и App Store ругает за `voip` в новых заявках.** OpenClaw iOS использует `audio` + `location` + `processing` background modes. App Review может попросить обоснование «зачем `audio` background, если приложение не плеер». Internal-preview не подаётся в Review, но при публикации это будет блокер.

5. **`NODE_BACKGROUND_UNAVAILABLE`** — не баг, а контракт. `camera.snap`, `screen.snapshot`, `talk` требуют foreground app state. Если телефон в кармане с заблокированным экраном — фотку сделать нельзя. Push-wake поднимет приложение на front, но юзер увидит сплеш-экран.

6. **Photo payload limit 5 MB base64.** Высокое разрешение (`maxWidth: 4000`) → recompress, потеря качества. Если нужен оригинал — сохраняй на телефоне в `~/Downloads/openclaw/` и шли только path в gateway, или используй S3-presigned-URL upload.

7. **Camera EXIF GPS не вырезается автоматически.** Если фото попадёт в логи Anthropic/OpenAI или в твой Postgres-history — там будет твой адрес. Делай `exiftool -gps:all= -overwrite_original` перед persist.

8. **Microphone в фоне на iOS — practически не работает.** AVAudioSession suspend через ~30 сек, как только app не frontmost. Continuous Talk Mode реалистично только при экране включённом или при активном Live Activity / Now Playing.

9. **Wake word не работает offline на Android.** Нет on-device модели до Issue #30447. Попытка «всегда слушать» сожжёт батарею за 4–5 часов и сольёт весь аудиопоток в gateway.

10. **Multiple users на одном gateway.** `openclaw devices` сейчас глобален, а не per-user (issue обсуждается в #47871). Если у Дмитрия и его команды — общий gateway, у всех видны все ноды. Воркэраунд — отдельный gateway-инстанс per user, либо ACL через `gateway.nodes.allowOnly` (если поле уже добавили) [VERIFY].

11. **iOS «Local Network» permission — отдельный от Bonjour.** Без него mDNS не сработает даже в одной WiFi. Юзер должен подтвердить системный диалог при первом запуске; если случайно отказал — Settings → OpenClaw → Local Network → On.

12. **TestFlight expires через 90 дней.** Каждые 90 дней приходит push «Beta expired», и нужно ре-инсталл. Pairing при этом сохраняется, если bundle id и device id те же.

13. **Несколько Android-приложений с похожим названием.** В Play Store можно нарваться на community-фейк (`andClaw` от не-openclaw автора, `ClawNode` от cb_studio). Проверяй publisher = `OpenClaw` или хотя бы реальный GitHub-link. [VERIFY: какой именно publisher официальный — текущая инфа конфликтует]

## 🧪 Верификация

**1. Pairing alive после reboot:**
```bash
# Перезагрузи телефон, подожди 30 сек:
openclaw nodes status --json | jq '.[] | select(.name=="iphone-15") | .online'
# Ожидание: true (через 5–10 сек после boot)
```

**2. Camera snap end-to-end:**
```bash
openclaw nodes camera snap --node iphone-15 --facing back > /tmp/snap.json
cat /tmp/snap.json | jq -r '.base64' | base64 -d > /tmp/snap.jpg
file /tmp/snap.jpg
# Ожидание: JPEG image data, JFIF, 1600x1200 (или близко)
```

**3. Location precision:**
```bash
openclaw nodes location get --node iphone-15 --precision high
# accuracy < 30 (метров) для GPS-fix outdoor
openclaw nodes location get --node iphone-15 --precision low
# accuracy ~ 500–1000 (метров) для cell-tower fix
```

**4. Foreground service alive 1+ час:**
- Положи телефон, экран погасни, подожди 1 час.
- `openclaw nodes status` → всё ещё online.
- Если offline — battery optimization не отключена или autostart выключен.

**5. End-to-end agent test:**
- В Telegram-боте (Блок 4): «Сфотографируй на заднюю камеру что передо мной и опиши».
- Бот должен ответить текстом-описанием в течение 10–15 сек.
- В логах gateway видим: tool_call camera.snap → multimodal LLM call → response.

**6. Privacy check:**
```bash
exiftool /tmp/snap.jpg | grep -i gps
# Ожидание (если pre-process включён): нет GPS-полей.
# Если есть — настрой stripping (см. лайфхак #10).
```

| Battery optimization + autostart (Android) | 5 мин (10 на Xiaomi) |
| Permissions + Local Network (iOS) | 3 мин |
| `gateway.nodes.allowCommands` конфиг | 2 мин |
| Тестовый запрос агенту | 5 мин |
| Дебаг при проблемах | +15–30 мин |
| **Итого Android, всё ОК** | **~30 мин** |
| **Итого iOS, TestFlight уже есть** | **~35 мин** |
| **iOS без инвайта** | блокер — пропускаем или ждём |

Реалистично — заложи 45 мин на первый телефон, 15 на второй (после первого повтор быстрее).
