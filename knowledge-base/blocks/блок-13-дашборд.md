# Блок 13: Дашборд (контроль и аналитика)

> Что: единая панель наблюдения за OpenClaw-агентом — траты, cron, память, чат, ошибки.
> Зачем: без дашборда «слепой полёт» — не видно ни счёта OpenRouter, ни падений cron, ни мусора в памяти.
> Время: 30–60 минут (минимум — встроенный Control UI через SSH-туннель за 10 мин; полный стек с Grafana и Tailscale — до 60 мин).

---

## Цель блока

Сделать так, чтобы у Дмитрия был **один URL** (например, `http://localhost:4000`), при открытии которого видно:

1. **Что бот делает прямо сейчас** — текущие задачи daemon-а, активный чат, последние tool-вызовы (sandbox/gateway/MCP).
2. **Сколько тратит** — траты по моделям (Claude Opus 4.7, Haiku 4.5, Sonnet 4.6, GPT-5, Gemini 2.5), токены, доллары за день/неделю/месяц.
3. **Что в памяти** — список файлов `MEMORY.md`, `SKILLS.md`, граф связей между задачами, размер контекстных файлов.
4. **Cron-задачи** — список зарегистрированных через `claw cron list` задач, последние запуски, статусы, кнопка «запустить вручную».
5. **Ошибки и алерты** — что упало, когда, почему (интеграция Sentry/PostHog опциональна).

Финал блока: открыл `localhost:4000`, увидел все 5 панелей, проверил что цифры обновляются реально-времени, можно вручную запустить cron-задачу и увидеть её лог.

## Лайфхаки и про-приёмы

1. **Не открывай дашборд наружу без mTLS.** Даже если кажется что «никто не знает URL» — боты-сканеры находят за минуты. Минимум — Tailscale (private network), максимум — Caddy с client-cert. Public-доступ через Cloudflare Tunnel + Access Policy с email-allowlist — третий вариант, но Tailscale удобнее.

2. **`claw ui --read-only` для шаринга.** Если нужно показать дашборд кому-то (например, бухгалтеру для отчёта по тратам) — есть флаг `--read-only`, который скрывает кнопки запуска cron, изменения памяти и т.д. Создаёт отдельный URL `http://localhost:4001/ro`.

3. **Двойной туннель: одновременно SSH + Tailscale.** SSH — резерв на случай если Tailscale упал (бывает после обновлений ядра на VPS). Конфиг ниже показывает оба сразу.

4. **Группируй траты по проектам через `CLAW_PROJECT` env var.** В `.env` каждого worktree задай `CLAW_PROJECT=cs-vision-v2.8` — дашборд автоматически разрежет графики по проекту. Без этого все траты в одной куче.

5. **Bookmark `localhost:4000/now`** — это live-страница «что бот делает прямо сейчас». Открой в отдельной вкладке Chrome, поставь auto-refresh (Tab Auto Refresh extension) на 5 сек — будет ambient-режим как Twitch-стрим.

6. **Алерт на дневной лимит трат** в дашборде. Идёшь в Settings → Spend Alerts → добавляешь `daily_limit: $5`, `notify: email`. Когда траты за день превысят лимит — придёт письмо. Спасает от runaway-loop в cron.

7. **`claw ui` + tmux на VPS.** Не запускай `claw ui` в foreground — потеряешь при отключении SSH. Запусти в `tmux new -s ui 'claw ui --port 4000'`, и потом просто `tmux attach -t ui` чтобы заглянуть в логи.

8. **MEMORY.md visualizer — режим graph.** В дашборде есть переключатель `List | Tree | Graph` для памяти. Graph рисует связи между файлами через mention-парсинг (`@MEMORY.md#section`). Полезно когда памяти стало много — видно что забыто.

9. **Cron Manual Run + Dry-Run.** Перед тем как запустить cron-задачу вручную из UI — нажми «Dry Run», она пройдёт без вызовов LLM (моки) и покажет что *планировала* сделать. Спасает от случайных $20 трат на тестах.

10. **mTLS-сертификат на iOS** — экспортируй `.p12` файл, отправь себе в Telegram saved messages, на iPhone открой → «Установить профиль» → дашборд работает в Safari. Гораздо удобнее чем VPN на телефоне.

11. **Grafana annotations из cron**. Каждый успешный/упавший cron автоматически постит annotation в Grafana через `claw cron annotate-grafana`. На графике трат видны вертикальные линии с подписью «cron: cleanup-memory ran 23:00». Корреляция с пиками — сразу видна.

12. **Sentry tag `claw.task_id`** — добавь в Sentry SDK `set_tag("claw.task_id", os.getenv("CLAW_TASK_ID"))`. Тогда в Sentry все ошибки группируются по задачам, можно увидеть «эта cron-задача падает 3-й раз подряд».

13. **PostHog session replay только для error-сессий** — иначе раздуешь quota. В настройках `replay.sample_rate=0.0`, но `replay.minimum_duration_ms=null`, и `replay.capture_on_exception=true`.

14. **Mobile bookmark на iOS Home Screen.** Открыл `https://claw.<твой-tailnet>.ts.net/mobile` в Safari → Share → Add to Home Screen. PWA-режим, fullscreen, выглядит как нативное приложение. mudrii dashboard поддерживает PWA с v2.2.

15. **Backup дашборд-конфига в git.** Все панели Grafana и mudrii — это JSON. Положи `~/.claw/ui/dashboards/*.json` в git-репо `claw-config`, и при переезде на новый VPS восстановишь одной командой `claw ui import ./dashboards/`.

## Готовые команды и конфиги

### 1. Простейший SSH-туннель (минимум)

```bash
# На VPS (один раз):
claw ui --port 4000 --bind 127.0.0.1 &

# На локальной машине (Mac):
ssh -L 4000:127.0.0.1:4000 -N user@vps.example.com

# Открыть в браузере:
open http://localhost:4000
```

Флаги: `-L` — local forward, `-N` — не открывать shell (только туннель).

### 2. Persistent autossh через systemd (Linux user-unit) или launchd (macOS)

**Linux — `~/.config/systemd/user/claw-tunnel.service`:**

```ini
[Unit]
Description=Persistent SSH tunnel to OpenClaw dashboard
After=network-online.target
Wants=network-online.target

[Service]
Environment="AUTOSSH_GATETIME=0"
Environment="AUTOSSH_PORT=0"
ExecStart=/usr/bin/autossh -M 0 -N \
  -o "ServerAliveInterval 30" \
  -o "ServerAliveCountMax 3" \
  -o "ExitOnForwardFailure yes" \
  -o "StrictHostKeyChecking accept-new" \
  -L 4000:127.0.0.1:4000 \
  -L 9090:127.0.0.1:9090 \
  -L 3000:127.0.0.1:3000 \
  -i %h/.ssh/id_ed25519_claw \
  user@vps.example.com
Restart=always
RestartSec=10

[Install]
WantedBy=default.target
```

Запуск:
```bash
systemctl --user daemon-reload
systemctl --user enable --now claw-tunnel.service
systemctl --user status claw-tunnel.service
```

### 3. Tailscale ACL для дашборда

`/etc/tailscale/acl.json` (или через Admin Console):

```json
{
  "tagOwners": {
    "tag:claw-server": ["dmitriy@example.com"],
    "tag:claw-client": ["dmitriy@example.com"]
  },
  "grants": [
    {
      "src": ["tag:claw-client"],
      "dst": ["tag:claw-server"],
      "ip": ["4000", "3000", "9090"]
    }
  ],
  "ssh": [
    {
      "action": "check",
      "src": ["autogroup:member"],
      "dst": ["tag:claw-server"],
      "users": ["claw"],
      "checkPeriod": "12h"
    }
  ]
}
```

На VPS:
```bash
sudo tailscale up --ssh --advertise-tags=tag:claw-server
sudo tailscale serve --bg --https=443 --set-path=/dash http://127.0.0.1:4000
```

После — дашборд доступен по `https://vps.<твой-tailnet>.ts.net/dash` с любого устройства в твоей tailnet.

### 5. Caddy с mTLS (для public-доступа без VPN)

`/etc/caddy/Caddyfile`:

```caddy
claw.example.com {
    tls {
        client_auth {
            mode require_and_verify
            trust_pool file {
                pem_file /etc/caddy/clients-ca.pem
            }
        }
    }

    reverse_proxy 127.0.0.1:4001 {
        header_up X-Client-Cert {tls_client_certificate_pem}
        header_up X-Client-CN {tls_client_subject}
    }

    log {
        output file /var/log/caddy/claw-access.log
        format json
    }
}
```

Генерация клиентского сертификата:
```bash
# CA
openssl genrsa -out clients-ca.key 4096
openssl req -x509 -new -key clients-ca.key -days 3650 -out clients-ca.pem -subj "/CN=Claw-CA"

# Клиент (Mac/iPhone)
openssl genrsa -out dmitriy.key 4096
openssl req -new -key dmitriy.key -out dmitriy.csr -subj "/CN=dmitriy@example.com"
openssl x509 -req -in dmitriy.csr -CA clients-ca.pem -CAkey clients-ca.key -days 365 -out dmitriy.crt

# .p12 для импорта в Keychain / iOS
openssl pkcs12 -export -out dmitriy.p12 -inkey dmitriy.key -in dmitriy.crt -certfile clients-ca.pem
```

## Подводные камни

1. **`claw ui` по умолчанию слушает 0.0.0.0** в версиях <0.9.0 — обязательно проверь `--bind 127.0.0.1`. Иначе твой VPS-IP сразу попадёт в логи Shodan.

2. **OpenRouter Usage API rate-limit** — 60 req/min. Если ты включил polling каждые 5 сек — упрёшься. Используй WebSocket (`ws_enabled = true`) либо подними `poll_interval_seconds` до 30+.

3. **autossh без `ExitOnForwardFailure yes` молча умирает** в случае если порт уже занят (ты уже запустил туннель в другом терминале). Всегда добавляй этот флаг.

4. **Tailscale Funnel — это public**, в отличие от `tailscale serve`. Не путай. Funnel = открыто всему интернету через `*.ts.net` поддомен. Для дашборда используй именно `serve`.

5. **mudrii dashboard <2.0 не поддерживает OpenRouter API v3**, и ломается на новых ответах. Обнови до 2.3+.

6. **Grafana по умолчанию слушает 0.0.0.0:3000** и пускает с пустым admin/admin. Первый шаг после установки — поменять пароль И в `grafana.ini` поставить `[server] http_addr = 127.0.0.1`.

7. **Sentry `send_default_pii = true` сливает промпты** в Sentry. По дефолту в SDK для Python это False, но в openclaw-sentry до v2.18 было True — проверь явно.

8. **PostHog session replay** записывает ВСЁ что в DOM — включая API ключи если они отображаются в UI. Включай только если в дашборде нет открытых секретов, либо настрой `mask: true` для всех `.secret` классов.

9. **mTLS `.p12` пароль теряется на iOS если не вписать при установке профиля** — окно появляется один раз, если пропустишь — переустанавливай. Запиши пароль заранее.

10. **Caddy `client_auth` без `trust_pool` пускает любого** с самоподписанным сертом. Обязательно `mode require_and_verify` + `trust_pool`.

11. **`claw ui` хранит сессию в `~/.claw/ui/sessions.db`** (SQLite). Если у тебя несколько worktree с одним домашним — будут конфликты. Используй `--state-dir` per-worktree.

12. **Multi-agent view в mudrii** требует чтобы каждый daemon имел уникальный `claw.daemon_id` в config.toml. По умолчанию все = "default", и в multi-view все сливаются в один.

13. **Cron Manual Run из UI** не учитывает `CLAW_PROJECT` env var из крона — он берёт env процесса UI. Если запустить вручную — траты упадут не в тот проект. Workaround: `claw cron run <id> --env CLAW_PROJECT=...`.

14. **systemd user-unit не запускается при boot** если включить надо `loginctl enable-linger $USER`. Иначе твой autossh поднимется только когда ты залогинишься по SSH.

15. **`dashboard-pack` НЕ существует как ClawHub-скилл** (поправка аудита). Раньше в этом блоке предлагалось `claw skill add dashboard-pack` — не делай, slug пустой. Импортируй Grafana-дашборды вручную из mudrii-репо или собирай свои.

## Верификация

1. **Открыть `http://localhost:4000`** — должен открыться дашборд с заголовком "OpenClaw Control".
2. **Панель "Now"** — показывает «Daemon: idle» или «Running task: ...». Если daemon включён, послать тестовое сообщение через `claw chat send "ping"` → в дашборде должно появиться сообщение.
3. **Панель "Spend"** — показывает $X.XX за сегодня, хотя бы один пункт по модели. Если 0$ — daemon ничего не вызывал, проверь.
4. **Панель "Cron"** — список задач. Кликнуть на задачу → видно историю запусков, кнопка "Run Now". Нажать "Dry Run" → должен показать planned actions без реального вызова LLM.
5. **Панель "Memory"** — список MEMORY.md/SKILLS.md. Переключить вид на "Graph" → должна нарисоваться сеть связей.
6. **Панель "Logs/Errors"** — пусто или несколько строк. Намеренно вызвать ошибку: `claw debug raise-test-error` → ошибка должна появиться в дашборде в течение 5 сек.
7. **Mobile-тест** — открыть `https://claw.<tailnet>.ts.net/mobile` на iPhone в Safari → лейаут должен быть колоночный, читаемые цифры, без горизонтального скролла.
8. **Алерт-тест** — поставить `daily_spend_usd_limit = 0.001`, сделать вызов LLM → должен прийти email через 1–2 минуты.
9. **mTLS-тест** — попробовать открыть дашборд с устройства БЕЗ сертификата → должен вернуться 403/Browser cert dialog.
10. **Tailscale-тест** — выключить Tailscale → дашборд через `https://claw.<tailnet>.ts.net` недоступен. Включить → доступен в течение 10 сек.

## Реальная оценка времени

- **Минимум (только встроенный UI + SSH-туннель):** 10–15 минут.
- **Средний (+ autossh systemd + mudrii dashboard + траты):** 30–40 минут.
- **Полный стек (+ Tailscale + Prometheus + Grafana + Caddy mTLS + Sentry):** 60–90 минут (если первый раз — добавь ещё 30 минут на чтение Caddyfile/ACL Tailscale).
- **С нуля без знаний systemd/Tailscale/mTLS:** 2–3 часа.

Реалистичная оценка для Дмитрия (с уже работающим OpenClaw): **45 минут до зелёного чек-листа без Caddy mTLS**, +30 минут если нужен mTLS.
