# 🆘 AI-Консультант по OpenClaw

> **Открой НОВЫЙ чат AI** (НЕ тот где идёт установка!) → **скопируй промпт ниже** → задавай вопросы.
>
> Консультант — отдельный AI, который **только консультирует**. Он НЕ запускает SSH, НЕ меняет VPS, НЕ перезапускает daemon. Только читает базу знаний + интернет → даёт точный ответ за 30 секунд.

---

## 📋 Зачем нужен консультант

Главный AI в основном чате (где ты делаешь установку) — он **исполнитель**. Он работает по промптам и фиксит проблемы через изменения.

Когда ты застрял — главного AI **легко уговорить попробовать что-нибудь** («давай переустановим», «давай переключим канал»). Это и был наш 2-дневный ад.

Консультант в отдельном чате:
- ✅ Только читает (не меняет систему)
- ✅ Имеет доступ к 2 МБ ресерча и 7 known-issues
- ✅ Не унаследовал bias главного AI
- ✅ Скажет «не делай этого» если ты собираешься на минное поле
- ✅ Сходит в интернет если нет в базе

## 🎯 Как использовать

1. Открой **новый чат AI**:
   - В Antigravity: создай новую сессию (не в той же что главный AI настройки!)
   - Или Claude.ai в браузере (https://claude.ai)
   - Или ChatGPT (https://chat.openai.com)
2. Убедись что AI **видит папку** `comandos-claw-deck/knowledge-base/` (через File Access в Antigravity или скопировав отдельный нужный файл в чат)
3. Скопируй **весь блок** ниже (между ═══ ═══) и вставь в первое сообщение
4. AI ответит «готов, задавай вопрос»
5. Опиши свою проблему → получи ответ

Пользователь: «Я запустил openclaw onboard но мастер скипнул вопрос
про authentication и теперь бот молчит. В логах gateway closed 1008.»

Ты:

## Прямой ответ
Это известная ловушка №1 — onboard прошёл с auth=skip вместо auth=token.
Pairing-state не создался, CLI остался в read-only scope, любой запрос
к моделям падает в 1008. Фикс: переустановка через ручной onboard.

## Где это в KB
knowledge-base/known-issues/01-1008-pairing-required.md
> «На пустом VPS правильная последовательность: npm i -g openclaw →
> openclaw onboard (интерактивный, ОБЯЗАТЕЛЬНО) → openclaw doctor --deep»

## Точная команда
```bash
# Backup всего
TS=$(date +%s)
tar czf /tmp/openclaw-backup-$TS.tar.gz -C ~ .openclaw

# Снос pairing state
systemctl --user stop openclaw
mv ~/.openclaw ~/.openclaw.broken-$TS
npm uninstall -g openclaw
npm i -g openclaw

# Заново — ИНТЕРАКТИВНО, ты сам в Mac Terminal!
openclaw onboard
```

На вопросах:
- Authentication mode → token (НЕ skip!)
- Enable device-pair plugin → yes

## Что НЕ делать
- ❌ openclaw devices approve --token ... (admin token не пробьёт scope-слой)
- ❌ openclaw doctor --fix (это by-design observe-only для security gate)
- ❌ openclaw onboard --non-interactive (опять скипнет pairing)

## 🛡 Когда НЕ обращаться к консультанту

- ❌ «Сделай за меня установку» — это работа главного AI или ручного пути
- ❌ «Дай мой API-ключ MiniMax» — консультант твоих ключей не знает
- ❌ «Подключись к моему VPS» — консультант не работает с инфраструктурой
- ❌ «Перепиши промпт» — это работа Дмитрия

Консультант = **советник**. Не исполнитель.
