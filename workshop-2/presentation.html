Что такое «память» для бота?

Представь обычного бота как секретаря с тяжёлой амнезией: каждый раз когда ты к нему обращаешься — он не помнит ни кто ты, ни что вы вчера обсуждали. Каждое утро ты заново рассказываешь ему про свой бизнес, про клиентов, про проекты.

🧠
Аналогия из жизни
Память для бота — это как записная книжка + секретарь + поисковая система в одном. Записная книжка — где лежат факты. Секретарь — кто их сам выписывает из ваших разговоров. Поисковик — как ты потом эти факты находишь когда нужно.

Сегодня мы научим твоего бота всему этому. К концу — он будет помнить тебя как хороший личный ассистент после 5 лет работы.

Общая идея

4 слоя памяти. У каждого своя задача.

Когда ты пишешь «запомни X» — оно попадает сразу в несколько слоёв одновременно. Чтобы потом найти можно было разными способами: и быстро в блокноте, и через поиск, и через библиотеку.

1
📁
Workspace files
Блокнот на столе
Файлы которые бот видит ВСЕГДА — без поиска. Личность, твои предпочтения, активные клиенты.
Где: VPS, файлы
2
🧠
Mem0
Секретарь-стенограф
Слушает диалоги, сам выдёргивает факты, кладёт их в индекс. Не надо говорить «запомни».
Где: VPS, Mem0 SDK
3
🗄
Qdrant
Поисковик по мыслям
Ищет по смыслу — спросишь «когда видел Анну?» — найдёт даже если ты её называл «сестрой».
Где: VPS, Docker
4
📚
Obsidian (опц)
Твоя библиотека
Гайдлайны, базы клиентов, фреймворки. Бот читает оттуда — пишет в твоём стиле.
Где: Mac → GitHub → VPS

Слой 2 · разбираем

Mem0 = секретарь-стенограф

2

Auto-capture
Что это простыми словами
Mem0 — это невидимый секретарь, который слушает все ваши разговоры и сам конспектирует факты. Не надо говорить «запомни» — он услышал, понял, записал.
Пример из жизни
Ты сказал боту: «Анна, моя сестра, в августе родит, надо подарок». Mem0 услышал и сам выдернул 3 факта: Анна = твоя сестра, событие = роды, дата = август 2026. Ты их не диктовал — он сам.
Когда работает
В фоне. Каждое твоё сообщение проходит через Mem0 — он решает «есть ли тут факт?» и если да — сохраняет.
Зачем нужно
Через месяц спросишь «что у меня в семье в этом году?» — бот вспомнит про Анну и роды. Без явного «запомни».

Слой 3 · разбираем

Qdrant = Google по твоим мыслям

3
🗄
Vector DB
Что это простыми словами
Qdrant — это поисковик по твоим фактам. Только не по словам как Google, а по смыслу. Спросишь «с кем я обсуждал AI?» — найдёт даже если в записях ты называл это «нейронкой» или «искусственным интеллектом».
Чем отличается от обычного поиска
Обычный grep ищет по точным словам. Qdrant ищет по «близости смысла» — преобразует каждый текст в число (вектор) и сравнивает математически. «AI» и «нейронка» оказываются «рядом».
Где живёт
Внутри Docker-контейнера на твоём VPS. Ты с ней напрямую не работаешь — это техническая база под капотом для Mem0 и других.
Сколько стоит
Сама база — бесплатная (open-source). Эмбеддинги через OpenAI — ~$1/мес.

Две диагностические команды. Первая — что съело контекст. Вторая — реальный расход и cache hit rate.

Hot-reload молча НЕ работает для compaction. После любой правки openclaw.json — рестарт.

Раскладываем файлы по полочкам

Главная ошибка которую совершают 90% — пихать всё в один файл. Задачи в SOUL.md, предпочтения в AGENTS.md. Через месяц это невозможно отлаживать. Поэтому ставим правильную структуру СРАЗУ.

Часть 2 · 15 минут

Почему бот тупеет через 2 недели?

Каждое твоё сообщение бот добавляет в свой «рабочий стол». Через 100 сообщений стол завален, бот не может ничего найти. Без защиты — через 2 недели сессия раздувается до 2.9 МБ и бот молча перестаёт отвечать.

🗂
Аналогия из жизни
Представь стол секретаря. На него каждый час кладут новые документы. Никто не убирает старые. Через месяц стол завален в потолок — секретарь не может даже свою чашку кофе найти. Решение: каждые час уборщик сжимает старые документы в краткие конспекты, а самые ненужные выбрасывает.
2.9MB
Размер сессии без защиты через 2 недели

Это и есть «потолок завален документами». Бот пытается каждый раз обрабатывать ВСЁ — падает по timeout, ты не понимаешь почему.

4 механизма решают это:

2.1
Compaction
«Уборщик» — сжимает середину разговора в конспект.
2.2
memoryFlush
ПЕРЕД сжатием — записывает важное в дневник.
2.3
contextPruning
Выбрасывает старый «мусор» (tool-results).
2.4
continuation-skip
Не дублирует SOUL/USER в каждом ходе.

Что происходит за 1 ход бота

Ты говоришь: «Запомни — Иван CTO Acme, бюджет $50k»

Через месяц спросишь «кто такой Иван?» — бот мгновенно находит в MEMORY.md (Слой 1, без поиска). Спросишь «с кем у меня бюджеты выше $30k?» — Qdrant (Слой 3) вернёт всех клиентов с big-budget tag.

А если ты сам не сказал «запомни»?

Ты в разговоре проболтался: «Анна, моя сестра, в августе родит — надо подарок»

Это и есть «секретарь-стенограф». Ты не диктуешь — он сам конспектирует. Через год у тебя в Qdrant сотни фактов, которых ты бы вручную никогда не записал.

Глоссарий простыми словами

Если что-то непонятно

Чтобы ты не запоминал «непонятные слова» — все термины собрали с переводом «как для друга на кухне».

Embeddings
Превращение текста в число. Каждое слово → длинный набор цифр. Похожие смыслы → близкие числа.
Vector DB
База данных для чисел (тех самых embeddings). Умеет искать «похожие» — это и есть semantic search.
Compaction
Сжатие середины разговора. 100 сообщений → 1 абзац-конспект. Старое забыто, важное сохранено.
Hybrid search
Поиск двумя способами одновременно — по точным словам (BM25) + по смыслу (vectors). Лучшее из двух миров.
TTS / STT
TTS = текст→голос (бот говорит). STT = голос→текст (бот слышит, это Whisper из В1).
Cron
Расписание для VPS. «Каждый час делай это» / «Каждый понедельник в 10:00 делай то».
git-crypt
Шифровальщик файлов. Кладёшь токены в репо — они не читаются без ключа. Безопасно.
Reranker
Второй фильтр поиска. Qdrant выдал 50 результатов — reranker отсортировал по реальной релевантности, оставил топ-5.
