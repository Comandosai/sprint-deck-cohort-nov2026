ДОКАЗАТЕЛЬСТВО №1 · TELEGRAM · СКРИН

🧠 Память выжила /reset

⏱ ~3 минуты

Шаг 1. Если в течение В2 ты уже рассказывал боту про свою работу, семью, планы, клиентов — переходи сразу к Шагу 2. Если не успел — пошли боту 3 любых факта про себя (любые имена, цифры, даты — какие хочешь) и подожди 60 секунд, чтобы auto-capture сохранил их в Mem0+Qdrant.

Шаг 2. Сбрось контекст — отправь команду в свой Telegram-бот:

КОМАНДА В TELEGRAM-БОТ

/reset

Шаг 3. Подожди 5 секунд (бот должен подтвердить сброс) и отправь боту:

СООБЩЕНИЕ В TELEGRAM-БОТ

Перечисли все факты которые я тебе говорил о себе за этот спринт. Используй mem0_search чтобы достать их из своей долговременной памяти. Ответь буллетами, без воды.
📸 Скриншот №1 — что должно быть видно: твой /reset и подтверждение бота, твой вопрос «перечисли факты...», ответ бота со списком твоих собственных фактов (минимум 2-3 пункта). Это главное достижение В2 — память пережила полный сброс контекста (D.1, D.2 стандарта).

ДОКАЗАТЕЛЬСТВО №2 · TELEGRAM · СКРИН

🔍 Бот видит свою инфраструктуру памяти

⏱ ~1 минута

Отправь боту одно сообщение и сделай скрин его ответа.

СООБЩЕНИЕ В TELEGRAM-БОТ

Сделай мне технический отчёт о своей памяти, в 4-5 строк:

1. Сколько фактов у тебя в Mem0+Qdrant обо мне (используй mem0_count или mem0_search)
2. Какая стратегия compaction — mode, keepRecentTokens, maxHistoryShare (прочитай ~/.openclaw/openclaw.json, секция compaction)
3. Какая embedding-модель у Mem0 (text-embedding-3-small или другая — посмотри в конфиге Mem0)
4. Жив ли Qdrant: ответь "qdrant: up" если контейнер работает, иначе "qdrant: down"

Сухой отчёт, без вступления.
📸 Скриншот №2 — что должно быть видно: ответ бота с конкретными цифрами и значениями: количество фактов в Mem0, mode/keepRecentTokens из openclaw.json, имя embedding-модели, статус Qdrant (up). Это доказывает что инфраструктура памяти настроена и доступна боту изнутри.

ДОКАЗАТЕЛЬСТВО №3 · TELEGRAM · СКРИН

📦 Автобэкап реально пушится в GitHub

⏱ ~1 минута

Отправь боту команду — он сам прогонит git log и вернёт результат в чат.

СООБЩЕНИЕ В TELEGRAM-БОТ

Покажи мне последние 5 автокоммитов своего бэкапа: запусти у себя

cd ~/.openclaw && git log auto/cron --oneline -5

И отдельной строкой выведи URL приватного GitHub-репо в который ты пушишь:

git remote get-url origin

Никаких пояснений — только вывод обеих команд.
📸 Скриншот №3 — что должно быть видно: список из 5 коммитов с короткими хешами и сообщениями (видно что они сделаны cron'ом) + URL твоего приватного GitHub-репо. Это доказывает что cron реально запускается каждый час и реально пушит в GitHub.

ШАБЛОН СООБЩЕНИЯ — В СПРИНТ-БОТ @ai_comandos_bot
