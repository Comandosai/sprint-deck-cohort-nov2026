═══ ИСТОЧНИК ИСТИНЫ ═══

Финальный результат описан в standards/workshop-1-standard.md этого репозитория.
Прочитай его сейчас. Там ~30 критериев готовности с уровнями важности

Если есть knowledge-base/ — посмотри его README. Там 20 блоков исследований
+ known-issues/ с уже встретившимися проблемами и фиксами.

═══ РАЗДЕЛЕНИЕ ТРУДА (важно!) ═══

Мы работаем в гибридном режиме:
- ТЫ делаешь рутину: SSH-ключи, VPS hardening, npm установка, правка конфигов
- Я делаю интерактивный openclaw onboard САМ в Mac Terminal (TTY-мастер)

⛔ ТЫ НЕ ЗАПУСКАЕШЬ openclaw onboard ни в каком виде. Даже с --non-interactive
   --accept-risk — этот флаг скипает critical pairing-шаги, и бот ловит 1008
   ошибку scope. Мы это проверили на 2-дневной боли.

═══ РЕЖИМЫ РАБОТЫ — DIAGNOSE → FIX → VERIFY ═══

Никогда не пропускай DIAGNOSE ради быстрого FIX — это ломает систему.

🟢 DIAGNOSE — собрать факты
   Когда: что-то не работает или результат непонятен.
   Что делаешь:
   - Собираешь сырые артефакты (logs, doctor, status, sessions)
   - НЕ меняешь конфиги
   - НЕ перезапускаешь daemon
   В конце: «Найдена причина: X. План фикса: 1 шаг — Y».

🟡 FIX — одно конкретное изменение
   Когда: причина найдена в DIAGNOSE.
   Что делаешь:
   - Одно изменение (не три, не пять)
   - Минимально инвазивное (правка строки, а не переустановка)

🟢 VERIFY — проверить что помогло
   Когда: после FIX.
   Что делаешь:
   - Те же команды что в DIAGNOSE
   - Если не помогло — снова DIAGNOSE, не делай второй FIX подряд.

═══ ЗАПРЕЩЕНО ПО УМОЛЧАНИЮ ═══

Эти действия требуют ЯВНОГО разрешения от меня в чате
(«да, переключай канал» / «да, переустанавливай»). Без разрешения — НЕ делай:

- ❌ openclaw onboard (любой формы — это TTY-мастер для человека)
- ❌ Переключение канала OpenClaw (stable / edge / canary / nightly)
- ❌ openclaw upgrade без явной просьбы
- ❌ Переустановка OpenClaw (npm uninstall && npm install)
- ❌ Удаление или пересоздание ~/.openclaw/
- ❌ Изменение tools.profile (lite ↔ full)
- ❌ Отключение плагина device-pair (это причина 1008 в 90% случаев)
- ❌ Пересоздание Telegram-канала / нового бота через @BotFather
- ❌ Удаление sessions / профилей / channels

Если думаешь что нужно одно из этих — СПРОСИ:
«Я предлагаю [действие], потому что [причина из DIAGNOSE]. Подтверди?»

═══ АНТИ-ПАТТЕРН: action bias ═══

Если ты застрял на проблеме больше 2 итераций — СТОП.
Это значит ты лечишь симптомы, а не причину.

Признаки action-bias:
- Ты переключаешь настройки туда-сюда «попробуем вот так»
- Ты предлагаешь переустановить когда не понял что сломано
- Ты делаешь FIX до того как закончил DIAGNOSE
- Ты не показал мне сырые логи

Что делать:
1. ОСТАНОВИСЬ. Не делай больше изменений.
2. Скажи: «Я в action-bias, нужна сводка состояния. Запускаю DIAGNOSE».
3. Собери артефакты:
   bash -lc "systemctl --user status openclaw --no-pager | head -20"
   bash -lc "openclaw doctor --deep 2>&1 | tail -60"
   bash -lc "openclaw models status"
   bash -lc "openclaw devices list"
   bash -lc "openclaw channels list"
   bash -lc "openclaw logs 2>&1 | tail -80"
4. Покажи мне сырой вывод КАЖДОГО.
5. Только после этого — план фикса в 1 шаг.

═══ КРИТИЧНЫЕ НЮАНСЫ OpenClaw 2026.4.x ═══

1. SLUG МОДЕЛЕЙ CASE-SENSITIVE
   ✅ minimax/MiniMax-M2.7  (с заглавными M!)
   ❌ minimax/minimax-m2.7  (нижний регистр — модель не найдена)

2. PATH ЛОВУШКА В non-login shell
   npm prefix ~/.npm-global/bin виден ТОЛЬКО в interactive bash.
   Cron, systemd, ssh non-interactive — НЕ ВИДЯТ его.
   Решение: PATH в ТРИ файла: ~/.bashrc, ~/.profile, ~/.bash_profile.
   Для одноразовых SSH-команд используй: ssh ... 'bash -lc "openclaw ..."'

3. DEVICE-PAIR ПЛАГИН — НЕ ОТКЛЮЧАТЬ
   plugins.entries.device-pair.enabled = true (всегда!)
   Без него любой scope-upgrade упирается в 1008 navigationally.

4. FALLBACK ТОЛЬКО ДЕШЕВЛЕ PRIMARY
   Реальный кейс из community: $4200 за 63 часа из-за fallback на дороже.
   Primary = MiniMax M2.7 ($10/мес фикс) → fallback = DeepSeek-V4-Flash
   (~$1/мес pay-as-you-go). НЕ наоборот!

5. SCHEMA MISMATCH в openclaw.json
   Поля auth.profiles, voice.transcription, spending caps, premiumGuard в
   JSON НЕ ПРИМЕНЯЮТСЯ (engine их игнорирует). Это by-design в 2026.4.x.
   Не пытайся «починить» эти секции через правку JSON — настраивай через CLI:
   openclaw auth set / openclaw fallbacks add / openclaw aliases set.
