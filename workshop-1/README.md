## 🚀 Поток прохождения

### Этап 1 — Pre-flight (до старта В1)

Должно быть готово ДО первого промпта:

- [ ] VPS куплен (Hetzner/Beget/Timeweb/etc), Ubuntu 24.04
- [ ] IP и root-пароль на email от хостера
- [ ] Antigravity установлен + AI-плагин активен (Claude Code или Codex)
- [ ] 6+ API-ключей получены: MINIMAX, DEEPSEEK, OPENROUTER, GROQ, OPENAI, TELEGRAM_BOT_TOKEN, TELEGRAM_USER_ID
- [ ] Telegram-бот создан через @BotFather

### Этап 2 — Установить контекст для AI (1 минута)

1. Открой `comandos-claw-deck/` в Antigravity (если ещё не открыт)
2. В чате с AI вставь **`00-meta-prompt.md`** целиком
3. Жди ответа AI «готов, давай первый промпт»

### Этап 3 — Прогнать 11 промптов (40-90 минут)

Открывай **`01-prompts.md`** и иди по порядку:
- Промпт 1 → AI выполняет → проверяешь → пишешь «прошло»
- Промпт 2 → ...
- ... до Промпта 11

⚠️ Только в Промптах 3 и 4 нужно ВРУЧНУЮ подставить `<VPS_IP>` и `<ROOT_PASSWORD>`.

### Этап 4 — Self-check через Telegram (10 минут)

После Промпта 11 открывай **`02-self-check.md`**:
1. Создай файл `audit/reports/workshop-1-<дата>.md`
2. Копируй 8 запросов в Telegram-боту
3. Сохраняй ответы в файл

### Этап 5 — Независимый аудит (5 минут)

Открой **`03-audit.md`** и:
1. Открой **НОВЫЙ** чат AI в Antigravity
2. Вставь промпт аудитора
3. Получи отчёт с вердиктом

### Этап 6 — Доделки (если есть)

В зависимости от вердикта аудитора:
- 🎉 «Пройдено» → готов к В2
- 🟡 «Почти готово» → есть ⚠️, доделать в течение недели
- ❌ «Проблемы» → починить через основной AI и повторить self-check + audit

## 🆘 Если что-то идёт не так

### AI блокирует передачу секретов на VPS

Это нормально. AI должен **сгенерировать команду для твоего терминала** с подставленными значениями. Ты вставляешь в Antigravity Terminal (`View → Terminal`) и запускаешь сам.

Если AI не догадался — напиши ему явно:
> «Дай мне готовую команду для моего терминала с подставленными значениями из .env, я вставлю сам».

### MiniMax отвечает через DeepSeek (а должен через MiniMax)

Это значит **probe MiniMax падает**. Самые частые причины:
1. Неправильный регистр slug — должно быть `minimax/MiniMax-M2.7` (с заглавными)
2. Битый/просроченный ключ
3. Не оплачена подписка Coding Plan

Скажи AI: «MiniMax не работает, в логах модель = deepseek/v4-flash. Найди причину через openclaw models status --probe minimax/MiniMax-M2.7 и почини».

### Что-то ещё

Сначала ищи в `../docs/troubleshooting.md`. Если не помогло — описывай проблему в общем чате спринта.
