# 🔍 Self-check — собираем артефакты от бота

> После прохождения 11 промптов в Antigravity — пора **проверить что у тебя на самом деле**.
> Бот сам отдаст артефакты текущего состояния. Эти артефакты пойдут в аудитор.

---

## Логика

Промпт 11 в Antigravity — это **самоотчёт основного AI**. Он может ошибаться 
или приукрашивать. Чтобы получить **независимое** мнение — спрашиваем сам бот 
о его состоянии. Бот отдаёт сырые файлы и команды.

---

## Подготовка

1. Создай файл `audit/reports/workshop-1-<сегодняшняя_дата>.md` (например 
   `audit/reports/workshop-1-2026-11-01.md`)
2. Открой Telegram, иди в чат с твоим ботом
3. Копируй запросы по одному, отправляй боту, **сохраняй ответы в этот файл**

## 8 запросов для бота в Telegram

Копируй каждый блок целиком и отправляй боту в Telegram.
Ответ бота сохраняй в `audit/reports/workshop-1-<дата>.md` под соответствующим 
заголовком.

### Запрос 1 — конфиг

```
Прочитай свой ~/.openclaw/openclaw.json и пришли его мне ЦЕЛИКОМ как файл, 
без изменений и без комментариев. Если он большой — пришли как вложение или 
разбей на куски, главное чтобы я получил весь текст.
```

### Запрос 2 — статус моделей

```
Запусти openclaw models status и пришли мне ПОЛНЫЙ вывод без сокращений.
Покажи также какая сейчас primary, какие fallbacks, какие aliases.
```

### Запрос 3 — auth profiles

```
Запусти openclaw auth list и пришли мне полный вывод. 
Сколько профилей и какие именно.
```

### Запрос 4 — каналы

```
Запусти openclaw channels list и пришли вывод. 
Покажи также детали Telegram-канала (через openclaw channels show если поддерживается).
```

### Запрос 5 — doctor --deep

```
Запусти openclaw doctor --deep и пришли мне ВЕСЬ вывод, без сокращений.
Особенно важны строки с критическими ошибками если они есть.
```

### Запрос 6 — crontab и watchdog

```
Запусти crontab -l и ls -la ~/.openclaw/scripts/ — пришли вывод обоих команд.
Если есть watchdog.sh — покажи его содержимое (но скрой токены маскированием 
вида "TG_TOKEN=***...***" если нашёл).
```

### Запрос 7 — workspace файлы

```
Покажи мне содержимое:
- ~/.openclaw/SOUL.md
- ~/.openclaw/USER.md (если есть)
- ~/.openclaw/MEMORY.md (если есть)
- ~/.openclaw/AGENTS.md (если есть, на VPS — это операционные правила бота)
- ~/.openclaw/HEARTBEAT.md (если есть)
```

### Запрос 8 — последний ответ и системка

```
Запусти openclaw logs --tail 30 — пришли вывод. 
Какая модель использовалась в последних 3 ответах? 
Это minimax/MiniMax-M2.7 или fallback на deepseek?

Также пришли:
- systemctl --user status openclaw --no-pager
- ss -tlnp | grep 18789
- node --version
- which openclaw
- swapon --show
- sudo ufw status
- loginctl show-user clawd | grep Linger
- systemctl is-active fail2ban
```

## ⚠️ Важные моменты

### Бот не понимает запрос?

Если бот говорит «не могу выполнить эту команду» — возможные причины:

1. **`tools.profile != "full"`** — exec команды не разрешены
2. **`exec.ask: on`** — бот спрашивает подтверждение, скажи «да» или «выполни»
3. **Бот не имеет доступа к workspace** — проверь `workspaceAccess: rw` в config

В этом случае — попроси AI в Antigravity **поправить tools/exec настройки** и попробуй снова.

### Бот скрывает секреты?

Если бот не показывает содержимое токен-файлов — это **правильно**. Там должны 
быть только `tokenFile: ~/.openclaw/secrets/telegram.token`. Сам файл с токеном 
проверять не нужно.

### Бот шлёт «Working...» / progress messages?

Если в ответе помимо файла видны технические маркеры (`sessions_yield`, 
`Working...`) — **просто проигнорируй их**. Сохраняй только содержательную 
часть ответа.
